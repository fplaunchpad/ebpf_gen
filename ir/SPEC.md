# Keel — a proof-carrying intermediate representation for eBPF

**Version 0 (draft). Scope: arithmetic fragment (ARITHUNARY, ARITHBINREG,
ARITHBINIMM), straight-line programs.**

Keel (working name: the load-bearing backbone of a program) is a
frontend-agnostic IR for correct-by-construction eBPF. Programs written in
SMT-backed proof languages (F*, Dafny, Lean + lean-smt, Verus, or anything that
can print S-expressions) are emitted as Keel text; the Keel **certifier**
compiles them to eBPF bytecode plus a **proof certificate**; a small,
formally verified **checker** — destined for the kernel — validates the
certificate against the bytecode with no solver, no abstract interpretation,
and no fixpoint computation.

Design lineage: per-obligation identity and assert-as-primitive from
Silver/Viper; instructions-as-Hoare-triples and VC-generation-as-total-function
from Vale; the expression-arena and recomputed-conclusions wire discipline
pioneered by BCF (SOSP'25); TAL's typing-derivation-as-certificate.
Differences from each are catalogued in §11.

## 1. Goals and non-goals

Goals (v0):
- G1: a frontend emits **instructions + sparse assertions**; everything else
  (densification, proof synthesis) is the certifier's job.
- G2: load-time TCB = the checker only: expression arena management,
  substitution, syntactic equality, and a fixed rule table (~24 rules).
- G3: certificates cannot be transplanted or forged (§10).
- G4: the checker is a reference model implemented and **verified in F***;
  the future in-kernel C implementation must agree with it.

Non-goals (v0): control flow (jumps — v1), memory and pointers (the
type/region layer — v2), helper calls, Spectre/speculative safety,
functional-correctness claims beyond register-value refinements.

## 2. Concepts

- **Program**: a sequence of arithmetic instructions ending in `exit`.
- **Machine state**: registers `r0..r10`, each holding a 64-bit value or
  *uninitialized*. Formal semantics: `fstar/Ebpf.Semantics.fst` (M1), pinned
  to RFC 9669 and differentially tested against Linux 6.8.
- **Annotation**: a QF_BV formula over register values at a program point.
- **Checker state** at point *i*: a **binding** per register (an arena term
  giving its exact value as a function of symbolic inputs, or ⊥ =
  uninitialized) plus a set of **claim** atoms (proved facts).
- **Obligation**: an atom the certificate must justify at a specific point
  (from a frontend `@assert`, or generated by the checker in strict mode).
  Obligations are indexed in program order — every obligation has an identity.
- **Certificate**: per-instruction step records (most are zero-content
  "SP-exact" steps) + proofs for obligations and claims.

## 3. Text format (`.kir`)

```
; comments start with ';'
.keel 0                          ; format version
.mode strict                     ; strict | kernel (M1 semantics, C5-C9)
.prog
  mov64  r1, 255
  and64  r1, 0x0f
  rsh64  r1, 1
  @assert (bvule r1 (_ bv7 64))  ; obligation #0
  mov64  r0, r1
  exit
```

Instruction mnemonics: `{op}{32|64} dst, src` with op ∈ {add, sub, mul, div,
sdiv, mod, smod, and, or, xor, lsh, rsh, arsh, mov, neg}, `src` a register or
signed 32-bit immediate; `movsx{8,16,32}{32|64} dst, rs`; `le{16,32,64}`,
`be{16,32,64}`, `bswap{16,32,64} dst`; `exit`. Exactly M1's AST
(`fstar/Ebpf.Ast.fst`); `@assert` corresponds to M1's `Assert_` generalized
from interval bounds to formulas.

There is **no `@assume`** (§10). The entry state is fixed: all registers
uninitialized.

## 4. Formula language

Concrete syntax: a subset of SMT-LIB2 QF_BV, so any frontend that can print
S-expressions can bind, and any annotation can be pasted into stock Z3/cvc5
for debugging.

```
term  ::= (_ bvN 64) | (_ bvN 32)              ; constants
        | r0 .. r10                             ; register at this point
        | (bvadd t t) | (bvsub t t) | (bvmul t t)
        | (bvudiv t t) | (bvsdiv t t) | (bvurem t t) | (bvsrem t t)
        | (bvand t t) | (bvor t t) | (bvxor t t) | (bvnot t)
        | (bvshl t t) | (bvlshr t t) | (bvashr t t) | (bvneg t)
        | ((_ extract 31 0) t) | ((_ zero_extend 32) t) | ((_ sign_extend N) t)
        | (ite atom t t)
atom  ::= (= t t) | (distinct t t)
        | (bvule t t) | (bvuge t t) | (bvult t t)
        | (bvule (_ bvN 64) t)  ; etc. — atoms are comparisons over terms
phi   ::= atom | (and phi phi+)                 ; conjunctions only
```

Quantifier-free, `old`-free, disjunction-free (v0): an annotation is a
conjunction of atoms; the checker splits it into per-atom obligations.

**Semantics are exactly SMT-LIB semantics.** eBPF's deviations are expressed
structurally when the checker builds definition equations (§6), never by
reinterpreting operators:
- shifts mask the amount: `(bvshl a (bvand s #x...3f))` (`#x...1f` for 32-bit),
- unsigned division by zero yields 0 in eBPF but all-ones in SMT-LIB, so the
  definition is `(ite (= s 0) 0 (bvudiv a s))`; same shape for `bvsdiv`,
- `bvurem`/`bvsrem` by zero return the dividend in **both** SMT-LIB and eBPF —
  used directly,
- ALU32: `((_ zero_extend 32) (op32 ((_ extract 31 0) a) ((_ extract 31 0) b)))`,
- `bvsdiv INT64_MIN -1 = INT64_MIN` in both — used directly.

## 5. Obligations

At each point, obligations arise from:
1. `@assert φ`: one obligation per conjunct atom of φ.
2. **Strict mode** (M1 C6/C9), generated by the checker — never by the
   certificate:
   - `div/sdiv/mod/smod dst, rS` → obligation `(distinct rS 0)`;
   - `lsh/rsh/arsh{N} dst, rS` → obligation `(bvult rS N)`.
   In kernel mode these obligations are not generated (the definition
   equations already carry total ITE/masked semantics, per M1 C5/C8).
3. Universal structural checks (not obligations — checker-internal, no proof
   can override them): dst ≠ r10, operand registers bound (≠ ⊥), immediate
   div-by-zero and immediate oversized shifts rejected (M1 C1–C4, C7),
   `movsx32` only at width 64, `r0` bound at `exit`, no trailing code after
   `exit` (C15, C17).

## 6. Checker algorithm (the load-time TCB)

State: `bind : reg -> arena_index | ⊥`, `claims : set arena_index` (atoms),
`arena : append-only DAG of terms/atoms`.

Initialization: `bind(r) = ⊥` for all r; `claims = ∅`. Symbolic input
variables exist only for future extension points (v0 programs start fully
uninitialized, so all bindings are built from constants).

Per instruction, in one linear pass:
1. Resolve operands through `bind` (⊥ ⇒ reject). Enforce structural checks (§5.3).
2. Build the **definition term** `d` for the destination per §4 (e.g. for
   `div64 r1, r2`: `d = (ite (= B(r2) 0) 0 (bvudiv B(r1) B(r2)))`) and set
   `bind(dst) := arena(d)`. This is the **SP-exact** step — it consumes zero
   certificate bytes beyond its step tag.
3. Discharge any obligations pending at this point (§5.1–5.2): for each, the
   certificate must supply a **proof** (§7) whose final conclusion is
   syntactically the obligation atom (resolved through `bind`). Exception: if
   the atom is already in `claims` or is a reflexive/constant tautology the
   checker can decide by evaluation, a `HIT` step (1 byte + index) suffices.
4. Optional certificate-directed state edits:
   - `CLAIM(atom, proof)`: check the proof, add the atom to `claims`
     (this is the *weakening* step — how derived facts persist);
   - `FORGET(indices)`: drop claims / release dead arena refs (always sound;
     bounds memory).

Complexity: every step is O(size of the referenced terms); one pass; no
search. DoS hygiene: certificate size cap, arena indices strictly backward,
arities/widths checked on arena append, per-program arena node cap,
reject-by-default on any malformed record.

**Modes.** `strict` and `kernel` differ only in §5.2 obligation generation —
same rule table, same step kinds (M1's dual-mode design carried forward).

## 7. Proof system

A proof is a DAG of steps; each step is `(rule_id, premises: step indices,
params: arena indices / small ints)`. **Conclusions are not in the wire
format** — the checker recomputes each step's conclusion from the rule table;
any mismatch (arity, width, side condition, premise shape) rejects the
certificate. The final step's conclusion must equal the goal atom exactly
(syntactic equality on arena indices — the arena hash-conses, so syntactic
equality is index equality).

Judgment: `S ⊢ atom` where S = current checker state (bindings define
equations; claims are available facts).

### Rule catalog v0 (~24 rules)

Leaves:
| id | rule | conclusion (computed) | side condition |
|----|------|----------------------|----------------|
| 0x01 | `DEF r` | `(= r_binding_term r_binding_term)` — never needed directly; see note | — |
| 0x02 | `CLAIMED i` | the claim atom at index i | i ∈ claims |
| 0x03 | `EVAL_EQ t` | `(= t c)` where c = checker-evaluated constant | t ground (constants only) |
| 0x04 | `ULE_CONST c1 c2` | `(bvule c1 c2)` | c1 ≤ c2 (checker evaluates) |
| 0x05 | `NE_CONST c1 c2` | `(distinct c1 c2)` | c1 ≠ c2 |
| 0x06 | `ULE_REFL t` | `(bvule t t)` | — |

Equality plumbing:
| 0x10 | `SYMM p` | flip `(= a b)` → `(= b a)` |
| 0x11 | `TRANS_EQ p q` | `(= a b), (= b c)` → `(= a c)` |
| 0x12 | `CONG op p` | `(= a b)` → `(= (op a c) (op b c))` (params: op, side, other arg) |
| 0x13 | `EQ_ULE p` | `(= a b)` → `(bvule a b)` |
| 0x14 | `SUBST_ULE p q` | `(= a b), (bvule b c)` → `(bvule a c)` (and symmetric variants) |

ITE collapse (division definitions):
| 0x18 | `ITE_F p` | premise `(distinct s 0)` collapses `(ite (= s 0) x y)` → conclusion `(= (ite (= s 0) x y) y)` |
| 0x19 | `ITE_T p` | premise `(= s 0)` → `(= (ite (= s 0) x y) x)` |

Unsigned-bounds rules (each is an M1 `Ebpf.Interval` lemma reified;
premise terms must match the definition-term shapes of §4):
| 0x20 | `TRANS_ULE p q` | `ule a b, ule b c` → `ule a c` |
| 0x21 | `AND_LE_L` / 0x22 `AND_LE_R` | — → `(bvule (bvand a b) a)` (resp. b) |
| 0x23 | `SHR_LE` | — → `(bvule (bvlshr a s) a)` (any s, masked or not) |
| 0x24 | `SHR_BOUND p` | `ule a c` → `(bvule (bvlshr a k) (c >> k))`, k const < width |
| 0x25 | `MONO_ADD p q` | `ule a ca, ule b cb` → `ule (bvadd a b) (ca+cb)` | side: ca+cb no wrap |
| 0x26 | `MONO_MUL p q` | `ule a ca, ule b cb` → `ule (bvmul a b) (ca*cb)` | side: ca*cb no wrap |
| 0x27 | `MONO_SHL p` | `ule a c` → `ule (bvshl a k) (c << k)`, k const | side: c << k no wrap |
| 0x28 | `DIV_LE` | — → `(bvule (bvudiv a b) a)` |
| 0x29 | `DIV_BOUND p q` | `ule a c, uge b d` → `ule (bvudiv a b) (c / d)` | side: d ≥ 1 |
| 0x2a | `MOD_BOUND p q` | `ule b c, distinct b 0` → `ule (bvurem a b) (c - 1)` | side: c ≥ 1 |
| 0x2b | `ZEXT_BOUND` | — → `(bvule ((_ zero_extend 32) t) #x00000000ffffffff)` |
| 0x2c | `UGE_FROM_ULE_SWAP`, lower-bound variants of ADD/SUB/MUL/DIV (`MONO_*_LO`) | analogous |
| 0x2d | `NE_FROM_UGE p` | `uge a c` → `distinct a 0` | side: c ≥ 1 |

Note on `DEF`: definition equations are implicit — bindings ARE terms, so an
obligation like `(bvule r1 7)` resolves to `(bvule <def-term-of-r1> 7)` before
proof checking. Rules therefore pattern-match on definition-term *structure*
(e.g. `SHR_BOUND` requires the subject term to literally be `(bvlshr a k)`),
which is what makes conclusions computable without search.

Soundness contract: every rule has a machine-checked lemma
`premises valid ∧ side conditions ⟹ conclusion valid` in
`fstar/Ebpf.Proof.fst`, against the same formula semantics
(`Ebpf.Formula.sat`) used by the end-to-end theorem. The rule table is
versioned (`.keel N`); additions are backward-compatible.

### Worked example (ex_shift, strict mode)

After `mov64 r1, 255; and64 r1, 0x0f; rsh64 r1, 1`:
`bind(r1) = (bvlshr (bvand 255 15) (bvand 1 63))` — wait: immediate-shift
amounts are validated structurally (§5.3) and folded, so the definition is
`(bvlshr (bvand 255 15) 1)`. Obligation #0: `(bvule bind(r1) 7)`.
Proof (4 steps, ~30 bytes):
```
s0: AND_LE_R                      ; (bvule (bvand 255 15) 15)
s1: SHR_BOUND s0, k=1             ; (bvule (bvlshr (bvand 255 15) 1) 7)
```
Two steps. (`15 >> 1 = 7` computed by the checker.) Certificates for
constant-heavy code can equally use `EVAL_EQ` + `EQ_ULE`/`SUBST_ULE`.

## 8. Certificate binary format (sketch — final layout fixed in M2.1)

```
header  { magic "KEEL", u16 version, u16 mode, u32 insn_cnt,
          u32 arena_cnt, u32 step_rec_cnt, u32 proof_step_cnt }
arena   [ { u8 tag, u8 width, u16 aux, u32 args[arity(tag)] } ]   ; hash-consed DAG,
                                                                  ; args strictly backward
steps   [ per-insn record: u8 kind (SPEXACT | HIT | PROVE | CLAIM | FORGET)
          + kind-specific payload (obligation idx, proof range, atom idx) ]
proofs  [ { u16 rule_id, u8 n_premises, u8 n_params, u32 args[] } ]  ; conclusions omitted
```

Size model: SP-exact instructions cost 1 byte + shared arena nodes (which the
definition terms need anyway); proofs cost ~10 bytes/step with 2–5 steps
per obligation on the M1 corpus. Measured targets in M2.4.

## 9. Frontend binding contract

A frontend binding must print: the instruction list (any generator — e.g. M1's
correct-by-construction `Ebpf.Build` pipeline, or a plain assembler) and
`@assert` formulas in the §4 grammar. That is the whole contract; the
certifier does densification and proof synthesis. Bindings planned: F*
(`Ebpf.Emit.fst`, M2.3), Python (~100-line printer, M2.3); printer notes for
Dafny (via `{:extern}` shim over its SMT encoding of bv64), Lean (lean-smt
term printer), Verus (bit_vector-mode assertions) are documented here in v1
without implementations.

## 10. Threat model

- **No transplanting**: goals are constructed by the checker from the loaded
  bytecode (definition terms + mode obligations). A certificate for a
  different program yields different definition terms → conclusions no longer
  match goals → reject.
- **No forging**: conclusions are recomputed per rule; the final conclusion
  must equal the goal index. An invalid rule application is a structural
  mismatch; a valid proof of the wrong atom fails the final comparison.
- **No assumption smuggling**: no `@assume` exists in the wire format; entry
  state is checker-pinned; strict-mode obligations are checker-generated.
- **End-to-end theorem** (M2.1): for ALL byte strings c,
  `certcheck(prog, c) = accept ⟹ runx (sem_of mode) prog` is safe and every
  asserted atom holds at its point. Adversarial certificates are in the
  quantifier.
- **DoS**: size caps, backward-only indices, arity/width validation, single
  linear pass. Malformed ⇒ reject.
- Residual TCB: the F* metatheory toolchain (metatheory time only), the
  semantics-vs-kernel gap (mitigated by the M1 differential harness, kept
  green), extraction/C-rewrite fidelity (M4: reference-model testing, later
  Low* extraction).

## 11. Relation to prior systems

| | BCF (SOSP'25) | VEP (NSDI'25) | Vale | Silver/Viper | **Keel** |
|---|---|---|---|---|---|
| Kernel-side analysis | full verifier kept | symbolic exec re-run | n/a (F* is checker) | n/a (Z3) | **none — syntactic SP + rule table** |
| Certificate | on-demand SMT refutations | annotations + proof terms for every entailment | none (F* file) | none | **TAL derivation: SP-exact steps free, proofs only at claims/obligations** |
| Proof rules | ~50 incl. bitblast circuits | cvc5-style + sep-logic syntactic | n/a | n/a | **~24 domain-aware (MONO_*) — no bitblasting; nonlinear MUL is one step** |
| Verified checker | no (C) | no (rules in Coq, impl unverified) | n/a | n/a | **yes (F* reference = spec)** |
| Frontend surface | C via clang | annotated C | Vale DSL | AST API (JVM) | **insns + SMT-LIB assertions (printer-only bindings)** |

## 12. Roadmap

v1: conditional jumps — labels carry annotations (join = syntactic equality of
claimed state, per-path proofs); v2: memory — type/region layer beside QF_BV
(pointers = (region, offset); region rules structural, offsets stay QF_BV);
v3: helper contracts as pre/post spec library; termination measures.
