# Keel — a proof-carrying intermediate representation for eBPF

**Version 0 (draft 2, post adversarial review). Scope: arithmetic fragment
(ARITHUNARY, ARITHBINREG, ARITHBINIMM), straight-line programs.**

Keel (working name) is a frontend-agnostic IR for correct-by-construction
eBPF. Programs written in SMT-backed proof languages (F*, Dafny,
Lean + lean-smt, Verus — anything that prints S-expressions) are emitted as
Keel text; the Keel **certifier** compiles them to eBPF bytecode plus a
**proof certificate**; a small, formally verified **checker** — destined for
the kernel — validates the certificate against the bytecode with no solver,
no abstract interpretation, and no fixpoint computation.

Lineage: per-obligation identity and assert-as-primitive (Silver/Viper);
instructions-as-Hoare-triples and VC-generation-as-total-function (Vale);
expression-arena + recomputed-conclusions wire discipline (BCF, SOSP'25);
typing-derivation-as-certificate (TAL). Differences catalogued in §11.

## 1. Goals and non-goals

Goals (v0):
- G1: frontends emit **instructions + sparse assertions**; densification and
  proof synthesis are the certifier's job.
- G2: load-time TCB = the checker only: an append-only expression arena,
  index-equality comparisons, a memoizing ground evaluator, and a fixed rule
  table (~28 rules). No hashing, no search, no recursion beyond one memoized
  arena pass.
- G3: certificates cannot be transplanted or forged (§10).
- G4: the checker is a reference model implemented and **verified in F***;
  the in-kernel C implementation (M4) must agree with it.

Non-goals (v0): control flow (v1), memory/pointers (v2 — type/region layer
beside the value logic), helper calls, Spectre, signed-bound reasoning
(sdiv/smod/arsh/movsx results are havoc: nothing can be proved about them in
v0 beyond width bounds — sound, just imprecise; signed atoms arrive in v1),
functional-correctness claims beyond register-value refinements.

## 2. Concepts

- **Program**: a sequence of arithmetic instructions ending in `exit`.
- **Machine state**: registers `r0..r10`, each a 64-bit value or
  *uninitialized*. Formal semantics: `fstar/Ebpf.Semantics.fst` (M1), pinned
  to RFC 9669; differential target: **Linux 7.0 (Ubuntu 26.04)**, with 6.8
  retained as a version-drift data point.
- **Annotation**: a QF_BV formula over register values at a program point.
- **Checker state** at point *i*: a **binding** per register (arena term =
  its exact value as a function of program constants, or ⊥ = uninitialized)
  plus a set of **claims** (arena atoms proved so far).
- **Obligation**: an atom that must be justified at a point. Two classes,
  with different trust roles (§10):
  - **safety obligations** — generated by the checker from the bytecode
    (strict mode: divisor ≠ 0, shift < width). Non-negotiable.
  - **declared claims** — `@assert` atoms carried by the certificate. The
    checker proves them and reports the proven-claim list; consumers decide
    whether the claims they required are present. Kernel safety never
    depends on certificate-supplied atoms.
- **Certificate**: per-instruction step records (most are zero-payload
  "SP-exact" steps) + declared claims + proofs.

## 3. Text format (`.kir`)

```
; comments start with ';'
.keel 0                          ; format version
.mode strict                     ; strict | kernel — see §6 Modes
.prog
  mov64  r1, 255
  and64  r1, 0x0f
  rsh64  r1, 1
  @assert (bvule r1 (_ bv7 64))  ; claim #0
  mov64  r0, r1
  exit
```

Mnemonics (exactly M1's `fstar/Ebpf.Ast.fst`):
- `{add|sub|mul|div|sdiv|mod|smod|and|or|xor|lsh|rsh|arsh|mov}{32|64} dst, src`
  with `src` a register or immediate;
- `neg{32|64} dst` (unary — no source operand);
- `movsx{8|16}{32|64} dst, rs` and `movsx32_64 dst, rs` (register source only);
- `le{16|32|64} dst`, `be{16|32|64} dst`, `bswap{16|32|64} dst`;
- `@assert phi` (attaches to the state **after** the preceding instruction);
- `exit`.

Immediates: decimal or `0x` hex, range [-2^31, 2^31) written in two's
complement (`-1` ≡ `0xffffffff`). Width handling follows M1/RFC 9669: ALU64
immediates are sign-extended to 64 bits; ALU32 immediates are used as 32-bit
patterns (`fstar/Ebpf.Int.fst: imm64/imm32`).

There is **no `@assume`** (§10). Entry state: all registers uninitialized.

## 4. Formula language

Concrete syntax: a subset of SMT-LIB2 QF_BV. **Semantics are exactly
SMT-LIB semantics** — any annotation can be checked against stock Z3/cvc5.
All eBPF deviations are expressed structurally in definition terms (§5),
never by reinterpreting operators.

```
const ::= (_ bvDEC w)            ; canonical form, DEC decimal, w ∈ {32,64}
                                  ; parser also accepts #xHEX, normalized
term  ::= const | r0 .. r10
        | (bvadd t t) | (bvsub t t) | (bvmul t t)
        | (bvudiv t t) | (bvsdiv t t) | (bvurem t t) | (bvsrem t t)
        | (bvand t t) | (bvor t t) | (bvxor t t) | (bvnot t) | (bvneg t)
        | (bvshl t t) | (bvlshr t t) | (bvashr t t)
        | (concat t t)
        | ((_ extract hi lo) t) | ((_ zero_extend n) t) | ((_ sign_extend n) t)
        | (ite atom t t)
atom  ::= (= t t) | (distinct t t) | (bvule t t) | (bvult t t) | (bvuge t t)
phi   ::= atom | (and phi phi+)
```

Every term has a **width** (32, 64, or intermediate widths produced by
extract/concat); widths are checked at arena construction and pinned in the
encoding (§8). Quantifier-free, `old`-free, disjunction-free: an annotation
is a conjunction of atoms; the checker splits it into per-atom obligations.
No signed comparison atoms in v0 (§1 non-goals).

## 5. Definition terms (normative)

When instruction *i* writes `dst`, the checker rebinds `dst` to the
**definition term** below, where `A = B(dst)`, `S = B(src)` are the *current*
(pre-instruction) bindings, `K` an immediate after §3 width handling, and
`x32 = ((_ extract 31 0) x)`. All eBPF-vs-SMT-LIB divergences are explicit:

| instruction | definition term (64-bit form) |
|---|---|
| `add64/sub64/mul64/and64/or64/xor64 dst, src` | `(bvOP A S)` |
| `div64 dst, src` | `(ite (= S 0) (_ bv0 64) (bvudiv A S))` |
| `sdiv64 dst, src` | `(ite (= S 0) (_ bv0 64) (bvsdiv A S))` |
| `mod64 dst, src` | `(bvurem A S)` — SMT-LIB rem-by-0 = dividend = eBPF |
| `smod64 dst, src` | `(bvsrem A S)` — ditto; `bvsdiv INT64_MIN -1 = INT64_MIN` in both |
| `lsh64/rsh64/arsh64 dst, rS` | `(bvSHIFT A (bvand S (_ bv63 64)))` — masking explicit |
| `lsh64 dst, K` (K validated < 64) | `(bvshl A (_ bvK 64))` — unmasked constant |
| `neg64 dst` | `(bvneg A)` |
| `mov64 dst, src` | `S` (or the constant K) |
| `movsx8_64 dst, rs` | `((_ sign_extend 56) ((_ extract 7 0) S))` (16: 48/15..0) |
| `movsx32_64 dst, rs` | `((_ sign_extend 32) ((_ extract 31 0) S))` |
| any `OP32` | `((_ zero_extend 32) (op-at-32 A32 S32))` — computed at width 32, zero-extended |
| `movsx8_32 dst, rs` | `((_ zero_extend 32) ((_ sign_extend 24) ((_ extract 7 0) S)))` |
| `le{N} dst` | `((_ zero_extend 64-N) ((_ extract N-1 0) A))` |
| `be{N}/bswap{N} dst` | `((_ zero_extend 64-N) (concat ((_ extract 7 0) A) ... ((_ extract N-1 N-8) A)))` — byte-reversal via concat of extracts |

The certifier must emit exactly these shapes; the checker *constructs* them
itself (the certificate cannot influence them).

## 6. Checker algorithm (the load-time TCB)

Inputs: bytecode, certificate, **mode** (strict | kernel). Mode is **loader
policy** (e.g. a load attribute/sysctl), NOT certificate data: the
certificate header's mode field is informational and must equal the
requested mode or the load rejects. An "accepted" verdict means "safe under
the mode the loader requested", never "safe under the mode the certificate
chose".

State: `bind : reg -> arena_index | ⊥`; `claims : set of arena atom indices`;
`arena : append-only, bounds/width/arity-checked term DAG`.

Per instruction, one linear pass, in this order:
1. **Resolve + structural checks** (checker-internal; no certificate
   content can override): dst ≠ r10; operand registers bound (≠ ⊥);
   immediate divisor 0 and immediate shift ≥ width rejected; movsx32 only
   at width 64; `r0` bound at `exit`; no code after `exit`. (M1 C1–C4, C7,
   C15, C17.)
2. **Safety obligations** (strict mode only), resolved against the
   **pre-instruction** bindings — critically, *before* dst is rebound
   (dst may be the divisor/shift-amount register):
   - `div/sdiv/mod/smod{64} dst, rS` → obligation `(distinct S 0)`;
     32-bit forms → `(distinct ((_ extract 31 0) S) 0)`;
   - `lsh/rsh/arsh64 dst, rS` → `(bvult S (_ bv64 64))`;
     32-bit forms → `(bvult ((_ extract 31 0) S) (_ bv32 32))`.
   Each obligation is discharged by a certificate step: `HIT i` (the atom is
   already claims[i] — index equality) or `PROVE p` (a proof, §7, whose final
   conclusion index equals the obligation's atom index). Kernel mode
   generates no safety obligations (definition terms already carry total
   ITE/masked semantics — M1 C5/C8).
3. **Declared claims** at this point (`@assert` conjuncts, in program
   order): each is an arena atom supplied by the certificate, discharged by
   HIT or PROVE, then added to `claims` and appended to the **proven-claims
   list** reported on success.
4. **Rebind** dst per §5.
5. Optional `FORGET {indices}`: removes atoms from `claims` (never touches
   the arena — the arena is append-only for the whole check and freed
   wholesale afterwards). Always sound; bounds the claims set.

Complexity and DoS hygiene: all comparisons are u32 index equality; rule
applications do O(#premises) index work plus, for EVAL-class rules, ground
evaluation **memoized per arena node** (each node evaluated at most once per
program; total O(arena size)). The evaluator implements the totalized
SMT-LIB semantics explicitly (division-by-zero cases return the defined
values; the C implementation must guard before native div instructions).
Caps: certificate size, arena node count, claims set size, proof step
count — all rejected-by-default when exceeded. Arena args strictly
backward-referencing; widths and arities validated at append.

## 7. Proof system

A proof is a list of steps; each step is `(rule_id, premises: earlier step
indices, params: arena indices / small ints)`. **Conclusions are not on the
wire**: the checker computes each step's conclusion (an arena atom index,
appending to the arena if needed) from the rule table; any structural
mismatch — arity, width, premise shape, side condition — rejects. The final
step's conclusion index must equal the goal atom index.

Widths: every rule is width-generic but width-checked — all term operands of
a rule application must share a width w, and all constant computations in
conclusions and side conditions are performed at width w (e.g. `MONO_ADD`'s
`ca+cb` is computed mod 2^w and its no-wrap side condition is `ca + cb <
2^w` as unbounded integers).

### Rule catalog v0

Leaves:
| id | rule | conclusion | side condition |
|----|------|-----------|----------------|
| 0x02 | `CLAIMED i` | claims[i] | i ∈ claims |
| 0x03 | `EVAL_EQ t` | `(= t c)`, c = memoized evaluation | t ground |
| 0x04 | `ULE_CONST c1 c2` | `(bvule c1 c2)` | val(c1) ≤ val(c2) |
| 0x05 | `NE_CONST c1 c2` | `(distinct c1 c2)` | val(c1) ≠ val(c2) |
| 0x06 | `ULE_REFL t` | `(bvule t t)` | — |
| 0x07 | `ULT_CONST c1 c2` | `(bvult c1 c2)` | val(c1) < val(c2) |

Equality plumbing:
| 0x10 | `SYMM p` | `(= a b)` → `(= b a)` |
| 0x11 | `TRANS_EQ p q` | `(= a b), (= b c)` → `(= a c)` |
| 0x12 | `CONG op side p x` | `(= a b)` → `(= (op a x) (op b x))` (side selects arg position) |
| 0x13 | `EQ_ULE p` | `(= a b)` → `(bvule a b)` |
| 0x14 | `SUBST p q` | `(= a b)` + atom mentioning b at a param-selected position → same atom with a |

ITE collapse (division definitions; **guard identity is index equality**):
| 0x18 | `ITE_F p` | premise `(distinct s c)` where the subject term is literally `(ite (= s c) x y)` (same s, c indices) → `(= (ite (= s c) x y) y)` |
| 0x19 | `ITE_T p` | premise `(= s c)` (same indices as the guard) → `(= (ite (= s c) x y) x)` |

Unsigned-bounds rules (each reifies an `fstar/Ebpf.Interval.fst`-style
lemma; subject terms must match §5 definition-term shapes structurally —
that is what makes conclusions computable without search):
| 0x20 | `TRANS_ULE p q` | `ule a b, ule b c` → `ule a c` |
| 0x21 | `ULE_ULT p` | `ule a c1` → `ult a c2` | side: val(c1) < val(c2) |
| 0x22 | `AND_LE_L` / 0x23 `AND_LE_R` | → `(bvule (bvand a b) a)` (resp. `b`) |
| 0x24 | `SHR_LE` | → `(bvule (bvlshr a s) a)` — any s (masked or constant) |
| 0x25 | `SHR_BOUND p` | `ule a c` → `(bvule (bvlshr a k) (c >> k))`, k const, val(k) < w |
| 0x26 | `MONO_ADD p q` | `ule a ca, ule b cb` → `ule (bvadd a b) (ca+cb)` | side: no wrap at w |
| 0x27 | `MONO_MUL p q` | `ule a ca, ule b cb` → `ule (bvmul a b) (ca*cb)` | side: no wrap at w |
| 0x28 | `MONO_SHL p` | `ule a c` → `ule (bvshl a k) (c << k)`, k const < w | side: no wrap at w |
| 0x29 | `DIV_LE p` | `(distinct b 0)` → `(bvule (bvudiv a b) a)` — **nonzero premise required** (SMT-LIB `bvudiv a 0` = all-ones) |
| 0x2a | `DIV_BOUND p q` | `ule a c, uge b d` → `ule (bvudiv a b) (c / d)` | side: val(d) ≥ 1 |
| 0x2b | `MOD_BOUND p q` | `ule b c, distinct b 0` → `ule (bvurem a b) (c - 1)` | side: val(c) ≥ 1 |
| 0x2c | `ZEXT_BOUND` | → `(bvule ((_ zero_extend n) t) (_ bv(2^(w-n)-1) w))` |
| 0x2d | `MONO_ADD_LO p q r s` | `uge a la, uge b lb, ule a ha, ule b hb` → `uge (bvadd a b) (la+lb)` | side: **ha + hb no wrap at w** (upper premises exclude wraparound; lower bounds alone are unsound) |
| 0x2e | `MONO_MUL_LO p q r s` | analogous, side: ha*hb no wrap |
| 0x2f | `NE_FROM_UGE p` | `uge a c` → `(distinct a 0)` | side: val(c) ≥ 1 |
| 0x30 | `UGE_CONST c1 c2` | `(bvuge c1 c2)` | val(c1) ≥ val(c2) |

Soundness contract: every rule has a machine-checked lemma
(`fstar/Ebpf.Proof.fst`) against the formula semantics (`Ebpf.Formula.sat`)
used by the end-to-end theorem. The M1 `Assert_` interval pseudo-instruction
generalizes in M2 to formula asserts; the executable semantics of an
asserted atom is `sat`, and the end-to-end theorem states every proven claim
evaluates true at its point. The rule table is versioned (`.keel N`);
additions are backward-compatible.

### Worked example (ex_shift, strict mode)

After `mov64 r1, 255; and64 r1, 0x0f; rsh64 r1, 1` (immediate shift:
validated, unmasked): `bind(r1) = (bvlshr (bvand 255 15) 1)`.
Claim #0: `(bvule bind(r1) (_ bv7 64))`. Proof — two steps, ~20 wire bytes:
```
s0: AND_LE_R                      ; (bvule (bvand 255 15) 15)
s1: SHR_BOUND s0, k=1             ; (bvule (bvlshr (bvand 255 15) 1) 7)
```
(`15 >> 1 = 7` computed by the checker at width 64.) Constant-heavy code can
instead use `EVAL_EQ` + `EQ_ULE` + `SUBST`.

## 8. Certificate binary format (layout finalized in M2.1)

```
header  { magic "KEEL", u16 version, u16 mode,      ; mode must equal loader-requested
          u32 insn_cnt, u32 arena_cnt, u32 step_rec_cnt, u32 proof_step_cnt,
          u32 claim_cnt }
arena   [ { u8 tag, u8 width, u16 aux, u32 args[arity(tag)] } ]
          ; tags: CONST64 (2 words), CONST32 (1 word), REG, BVADD..BVNEG,
          ;   CONCAT, EXTRACT (aux = hi<<8|lo), ZEXT/SEXT (aux = n), ITE,
          ;   EQ, NE, ULE, ULT, UGE — fixed arity table; widths checked;
          ;   args strictly backward; NO hash-consing in the checker:
          ;   syntactic equality IS index equality, and it is the
          ;   certifier's job to share nodes (unshared duplicates only
          ;   bloat the certificate and fail to connect proofs — never
          ;   unsound)
claims  [ { u32 insn_idx, u32 atom_idx } ]          ; declared @assert atoms
steps   [ per-insn: u8 kind (SPEXACT | HIT | PROVE | FORGET) + payload ]
proofs  [ { u16 rule_id, u8 n_premises, u8 n_params, u32 args[] } ]
```

Size model: SP-exact instructions cost ~1 byte + arena nodes their
definition terms need anyway; proofs ~10 bytes/step, 2–5 steps per
obligation on the M1 corpus. Measured in M2.4.

## 9. Frontend binding contract

A binding prints: the instruction list and `@assert` formulas in the §4
grammar. Nothing else — densification and proof synthesis are the
certifier's. Bindings in M2.3: F* (`Ebpf.Emit.fst`), Python (~100-line
printer). Printer notes for Dafny/Lean/Verus documented in v1.

## 10. Threat model

- **No transplanting**: definition terms and safety obligations are
  constructed by the checker from the loaded bytecode. A certificate for
  different bytecode yields different definition terms → proof conclusions
  no longer match goals → reject.
- **No forging**: conclusions recomputed per rule; final conclusion must
  index-equal the goal. Invalid application = structural mismatch; valid
  proof of the wrong atom fails the final comparison.
- **No assumption smuggling**: no `@assume`; entry state checker-pinned;
  safety obligations checker-generated; mode is loader policy (§6), not
  certificate data.
- **Claims are not safety**: `@assert` atoms are certificate-declared. A
  malicious certificate can *omit* claims, never fake them; consumers
  compare the reported proven-claims list against what they require.
- **End-to-end theorem** (M2.1): for ALL byte strings c,
  `certcheck(prog, c, mode) = accept ⟹` execution is safe under
  `sem_of mode` and every reported claim atom evaluates true at its point.
  Adversarial certificates are inside the quantifier.
- **DoS**: caps on certificate size / arena / claims / proof steps;
  backward-only indices; arity+width validation at append; memoized
  evaluation (O(arena) total); single linear pass; totalized evaluator
  (no native div traps); reject-by-default on malformed input.
- Residual TCB: F* metatheory toolchain (metatheory time only); the
  semantics-vs-kernel gap (mitigated: M1 differential harness, target
  kernel 7.0, kept green); extraction/C-rewrite fidelity (M4:
  reference-model testing, later Low* extraction).

## 11. Relation to prior systems

| | BCF (SOSP'25) | VEP (NSDI'25) | Vale | Silver/Viper | **Keel** |
|---|---|---|---|---|---|
| Kernel-side analysis | full verifier kept | symbolic exec re-run | n/a (F* is checker) | n/a (Z3) | **none — syntactic SP + rule table** |
| Certificate | on-demand SMT refutations | annotations + proofs for every entailment | none (F* file) | none | **TAL derivation: SP-exact steps free; proofs only at obligations/claims** |
| Proof rules | ~50 incl. bitblast circuits | cvc5-style + sep-logic | n/a | n/a | **~28 domain-aware; no bitblasting; nonlinear MUL bound = one step** |
| Verified checker | no (C) | no (rules in Coq; impl unverified) | n/a | n/a | **yes (F* reference = spec)** |
| Frontend surface | C via clang | annotated C | Vale DSL | AST API (JVM) | **insns + SMT-LIB assertions (printer-only bindings)** |

## 12. Roadmap

v1: conditional jumps — label annotations, per-path proofs, join = index
equality of claimed states; signed comparison atoms + signed-bound rules.
v2: memory — type/region layer beside QF_BV (pointers = (region, offset);
region rules structural; offsets stay QF_BV). v3: helper contracts;
termination measures.
